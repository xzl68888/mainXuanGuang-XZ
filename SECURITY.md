# # 安全策略 (Security Policy)

## 支持的版本

目前只有最新重构的 2.x 版本获得官方安全更新与技术支持。

| 版本 | 支持状态 |
| :--- | :--- |
| 2.x  | ✅ 支持中 (当前主线) |
| 1.x  | ❌ 不再维护 |

---

## 安全特性

### 🔐 核心加密
- **AES-256-GCM** 端到端加密（E2EE）。
- 会话密钥完全在客户端本地生成，服务器不留存。
- 所有聊天消息在离开客户端传输前即完成加密。

### 🛡️ 漏洞防护措施
- ✅ **路径遍历攻击防护**：严格限制静态文件访问边界。
- ✅ **XSS 攻击防护**：客户端对所有输入强制执行 `escapeHtml()` 转义。
- ✅ **消息长度限制**：单条消息最大限制 2000 字符。
- ✅ **TTL 范围限制**：阅后即焚时间严格限制在 5 至 300 秒之间。
- ✅ **准入长度限制**：用户名限制 20 字符，房间名限制 30 字符。

### 💻 Electron 安全配置
- ✅ `contextIsolation: true` （开启上下文隔离）
- ✅ `nodeIntegration: false` （禁用渲染进程 Node.js 集成）
- ✅ `sandbox: true` （强制开启沙箱模式）
- ✅ `webSecurity: true` （启用同源策略及安全检查）

## 报告漏洞

如果你在本项目中发现了任何潜在的安全漏洞，请**不要**直接创建公开的 GitHub Issue，以免漏洞细节泄露。请通过以下流程安全地向我们报告：

1. **私密报告**：请将漏洞详情、复现步骤及潜在影响发送邮件至项目维护者（或通过 GitHub 的 "Report a vulnerability" 私密功能提交）。
2. **响应周期**：我们将在接收到报告后的 **48 小时内** 给予初步回应，并评估漏洞风险。
3. **修复与致谢**：一旦漏洞被确认，我们会在私密分支中进行修复，并在发布补丁版本（如 v2.0.1）后公开说明，同时对报告者致以谢意。
